The 10-second flow. Footage is focused on load — the assistant types "120", the whole catalog prices itself, they click the material the customer asked about, and the headline number is ready while the customer is still talking. Gates and tear-down are one click each when they come up. No project exists, nothing was saved, nothing to clean up.
The range IS the quote. The headline shows −5% / +10% of the engine's real price, rounded out to clean $100s — the qualifying range the assistant already gives, now computed instead of eyeballed. The exact figure rides underneath in small print for whoever promotes it to a project.
Defaults are invisible, not secret. Labor $12/ft and margin 30% (verified: every live project uses exactly these), tear-down/dump $3+$3, permit $800. They don't appear on this screen because the assistant never changes them, not because they're hidden (you said full trust, one PIN). One build note: labor/margin are per-project fields today with no global default — Quick Quote adds default_labor_cost_ft and default_margin to settings, editable on the Prices screen, seeded 12 / 30%.
Save as project = the promote. Name + address (both optional in the moment) → creates a real project with one measurement named "Phone estimate", the chosen material on the board and set active, gates attached, permit/tear-down carried over — and snapshots prices per the effective-date model. The scratch pad then resets for the next call. Otherwise nothing is ever written; state lives in localStorage so a refresh mid-call loses nothing.
Decided: gates auto-match the material. The walk/double gate price follows the selected fence family automatically (Wood fence → Wood gate, Cypress → Cypress gate…). Mixed combos (wood fence, aluminum gate) go through the full project screen.
Decided: tear-down assumes the full run. The toggle bills tear-down + dump on ALL the footage (take-down mirrors footage, same as the phone default). Partial tear-downs are a project-screen detail, not a qualifying-call detail.
